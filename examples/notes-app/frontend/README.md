# notes-app frontend — iteration 3

Pure [assistant-ui](https://www.assistant-ui.com) + [shadcn/ui](https://ui.shadcn.com) shell wired to the **live notes-app FastAPI backend** via the canonical AG-UI streaming protocol and tenant-scoped `/threads` CRUD endpoints.

This iteration deliberately uses **only** primitives that ship with assistant-ui — no hand-rolled chat / thread / message / composer components. Streaming is owned by `@assistant-ui/react-ag-ui`; thread persistence is owned by `useRemoteThreadListRuntime` + our `RemoteThreadListAdapter` (`src/lib/thread-list-adapter.ts`).

## Stack

- Next.js 16 (App Router, Turbopack) + React 19 + TypeScript (strict)
- Tailwind v4 + shadcn/ui (`nova` preset, neutral base)
- `@assistant-ui/react` + `@assistant-ui/react-markdown`
- `@assistant-ui/react-ag-ui` + `@ag-ui/client` for the streaming runtime
- `assistant-stream` for the (currently no-op) `generateTitle` stream
- `next-themes` for the dark-mode toggle

## Install & run

Start the backend first (see `../backend/README.md`):

```bash
cd ../backend && OPENROUTER_API_KEY=... uv run uvicorn notes_app.main:app --reload --port 8000
```

Then the frontend:

```bash
pnpm install
cp .env.local.example .env.local   # override NEXT_PUBLIC_API_URL / NEXT_PUBLIC_TENANT_ID if needed
pnpm dev                            # http://localhost:3000 (falls back to next free port)
```

Configuration:

| Env var | Default | Purpose |
| --- | --- | --- |
| `NEXT_PUBLIC_API_URL` | `http://localhost:8000` | Backend base URL — `/threads` CRUD + streaming endpoint |
| `NEXT_PUBLIC_TENANT_ID` | `00000000-0000-0000-0000-000000000001` | Sent on every request as `X-Tenant-Id`; single-tenant dev for now |

Production build:

```bash
pnpm build
pnpm start
```

## Layout / routes

- `/` — single page. Layout is `AssistantSidebar`:
  - **Left panel**: header (title + theme toggle) and `ThreadList`
  - **Right panel**: `Thread` (welcome, suggestions, message list with markdown, composer with attachments, action bar, branch picker)
  - Resizable divider courtesy of shadcn `resizable`

## Where stuff lives

| Path | Purpose |
| --- | --- |
| `src/components/runtime-provider.tsx` | Wires `useRemoteThreadListRuntime` + `useAgUiRuntime` against the FastAPI backend. Subclasses `HttpAgent` to rewrite the POST URL per-thread (`/threads/{id}/messages?protocol=ag-ui`). |
| `src/lib/thread-list-adapter.ts` | `RemoteThreadListAdapter` mapped onto the backend's tenant-scoped `/threads` CRUD. `generateTitle` is a no-op empty stream (see RETRO). |
| `src/components/theme-provider.tsx`, `theme-toggle.tsx` | `next-themes` wiring |
| `src/components/assistant-ui/` | Generated by `pnpm dlx shadcn@latest add https://r.assistant-ui.com/...` — vendored, owned, do not edit unless you mean it |
| `src/components/ui/` | shadcn primitives (button, dialog, resizable, tooltip, …) |
| `src/app/layout.tsx`, `src/app/page.tsx` | Root layout + the single page that composes everything |

## What's exercised

- Thread list: create / switch / archive / delete (via `ThreadListPrimitive`)
- Welcome screen with suggestions
- Composer with attachment dropzone + add-attachment button (UI present; no upload adapter wired)
- Streaming assistant message with smooth text animation, copy / reload / branch actions, stop button
- Markdown rendering with `remark-gfm`
- Dark mode (system / light / dark)
